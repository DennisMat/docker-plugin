The duration (in seconds) for which a cloud or template will be removed from service in the event that a provisioning error is encountered:
  • If an error is encountered while provisioning a container from a Docker Template, that template will be ignored for this duration, allowing Jenkins to try using other (hopefully working) templates (potentially on other clouds, if configured).
  • If an error is encountered while communicating with the Docker service that is not directly related to a specific Docker Template then the cloud itself will be ignored for this duration, effectively taking out all templates within that cloud for this duration.
General guidance:
High durations (e.g. minutes) work well where there are multiple clouds/templates configured (i.e. Jenkins is not limited to just one source of build executors), allowing other cloud/templates to provide a means of running builds.
Medium durations (e.g. one minute) work well where the Docker service serves more than one Jenkins server as this will prevent Jenkins from "spamming" the shared Docker resource with failing requests.
A low (or zero) duration should only be used where the Docker service is not shared, not always reliable, and when there are no alternatives to the configured Docker Templates. i.e. when Jenkins should simply re-try failing templates until they work.

Defaults to 300 (5 minutes) if not set (or set to an invalid value). A value of zero results in no auto-disablement.
